Le **Canevas** est l'ensemble des notes qui definissent qui tu es pour Scapin.
Ces informations sont injectees dans chaque analyse IA pour personnaliser les decisions.

> **Etymologie** : En commedia dell'arte, le *canevas* est le scenario de base qui guide
> l'improvisation des acteurs. Ici, il guide l'improvisation des valets IA.

## Emplacement

Le Canevas se trouve dans le dossier **Memoires > Canevas** (icone ðŸ“œ).

Ce dossier est epingle en premier dans la liste des dossiers et affiche un badge "Contexte".

## Fichiers du Canevas

| Fichier | Contenu | Obligatoire |
|---------|---------|-------------|
| **Profile.md** | Identite, contacts, preferences personnelles | Oui |
| **Projects.md** | Projets actifs classes par priorite | Oui |
| **Goals.md** | Vision, objectifs pro/perso, principes | Oui |
| **Preferences.md** | Style de communication, preferences systeme | Non |

## Indicateur de completude

Le statut du Canevas est visible a deux endroits :

1. **Sidebar** (en bas) : Un indicateur permanent avec l'icone ðŸ“œ
   - Vert : Canevas complet
   - Orange : Canevas partiel
   - Rouge : Canevas incomplet

2. **Page d'accueil** : Un widget d'alerte s'affiche si le Canevas est incomplet,
   avec un lien direct pour le completer.

## Comment bien rediger

### Profile.md

- Ton nom complet et surnom
- Adresse email principale
- Role professionnel actuel
- Preferences de communication (horaires, canaux)
- Informations personnelles pertinentes (famille, lieux)

### Projects.md

- Liste tes projets par priorite (Haute, Active, Personnel)
- Pour chaque projet : nom, description courte, personnes cles
- Mets a jour regulierement quand les priorites changent

### Goals.md

- Ta vision a long terme
- Objectifs professionnels actuels
- Objectifs personnels
- Principes de decision (ce qui guide tes choix)

### Preferences.md (optionnel)

- Style de reponse prefere (concis, detaille)
- Outils utilises (OmniFocus, Calendrier...)
- Preferences de notification
- Langues de communication

## Bonnes pratiques

1. **Mettre a jour regulierement** â€” Un Canevas obsolete degrade les analyses
2. **Etre specifique** â€” "Je travaille chez Eufonie" > "Je travaille"
3. **Inclure les relations** â€” Les personnes frequentes, leurs roles
4. **Eviter les informations sensibles** â€” Pas de mots de passe ou secrets

## Impact sur les analyses

Le Canevas influence directement :

- **L'action recommandee** : Archiver, repondre, creer une tache...
- **La categorisation** : Travail vs Personnel
- **Le contexte enrichi** : Relations avec les expediteurs
- **Les notes proposees** : Liens avec projets existants

Un Canevas bien rempli permet a Scapin de prendre des decisions plus pertinentes
et personnalisees pour toi.

## Difference avec le Briefing

| Concept | Description |
|---------|-------------|
| **Canevas** | Contexte *permanent* â€” qui tu es, tes projets, tes objectifs |
| **Briefing** | Resume *ponctuel* â€” le briefing matinal ou pre-reunion |

Le Canevas est injecte dans chaque analyse IA.
Le Briefing est un rapport genere a la demande.
